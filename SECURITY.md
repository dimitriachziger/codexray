# Security Policy

## Supported versions

Security and privacy fixes are provided for the latest release on the `main`
branch.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability, privacy leak,
unsafe redaction, or a report containing sensitive rollout data.

Use GitHub's private vulnerability reporting for this repository:

<https://github.com/dimitriachziger/codexray/security/advisories/new>

Include a concise description, affected version, impact, reproduction steps,
and a minimal synthetic example. Do not upload an unredacted rollout. If real
data is essential, first describe why and wait for a secure exchange method.

You should receive an acknowledgement within seven days. Confirmed reports will
be investigated privately, with status updates provided through the advisory.
Disclosure timing will be coordinated after a fix is available.
